# 工具目录

本目录存放项目执行阶段的本地工具，不混入正文、章纲和设定文件。

## 正文字数统计

统计第一卷正文草稿：

```bash
python3 05工具/count_draft_words.py
```

默认统计 `03正文草稿/` 下符合 `001_章名.md` 这类命名的章节文件，并排除 `000_` 开头的说明文件。

输出字段：

| 字段 | 说明 |
|------|------|
| 正文字数 | 去掉 Markdown 标题、代码块和空白后，按中文字符 + 英文/数字词计数 |
| 含标点字符 | 去掉 Markdown 标题、代码块和空白后，保留标点的可见字符数 |
| 状态 | 默认 3000-3500 为达标，3501-4000 为可接受，超过 4000 为偏长 |

常用参数：

```bash
# 调整理想上限和可接受上限
python3 05工具/count_draft_words.py --min 3200 --ideal-max 3600 --max 4200

# 输出 JSON，便于后续自动化处理
python3 05工具/count_draft_words.py --json

# 只统计某个正文目录
python3 05工具/count_draft_words.py --draft-dir 03正文草稿/01_第一卷_青石坊市
```

正文流程中，每章写完并自审时运行一次该工具，用它判断章节篇幅是否明显偏离目标。当前写作仍以 3000-3500 字为理想区间，4000 字以内视为可接受；篇幅判断只作为执行辅助，不替代章节功能审阅。
